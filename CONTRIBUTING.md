# Contributing to sdk-python

Thank you for your interest in contributing to the Lineage Foundation sdk-python SDK.

## How to Contribute

1. **Open an issue** – For bugs, features, or questions, open an issue in this repository.
2. **Fork and clone** – Fork the repo and clone it locally.
3. **Create a branch** – Use a descriptive branch name (e.g. `feat/add-x`, `fix/issue-y`).
4. **Make changes** – Follow the code style in this repo (see [.editorconfig](.editorconfig)).
5. **Run tests** – Ensure tests pass before submitting.
6. **Submit a PR** – Open a pull request against `main`. Describe your changes clearly.

## Code Style

- Follow the [EditorConfig](.editorconfig) settings for this repository.
- Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages:
  - Format: `<type>[optional scope]: <description>`
  - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`
  - Example: `feat(api): add wallet creation method`

## Pull Request Process

- PRs require review before merge.
- Squash commits when merging.
- Ensure CI passes (if configured).

## Questions?

Open an issue or contact the maintainers via [lineage.foundation](https://lineage.foundation).
